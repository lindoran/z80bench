# z80bench — Disassembler Interface Spec

This document defines the interface between `disasm.c` and the rest of the library.
The internals of `disasm.c` may evolve (subprocess → linked decode) but the types
and function signatures here are stable.

z80dasm supplies the mnemonic and operand text. `disasm.c` supplies everything
else: byte data (from the ROM blob), address tracking (from the size table), and
data region formatting. Nothing is recomputed that z80dasm already provides.

---

## The DisasmLine Struct

The fundamental unit of output from `disasm.c`. Every instruction, data pseudo-op,
and gap byte is represented as one `DisasmLine`.

```c
#define DISASM_MNEMONIC_MAX   16
#define DISASM_OPERANDS_MAX   64
#define DISASM_LABEL_MAX      64
#define DISASM_COMMENT_MAX   256
#define DISASM_XREF_MAX      256
#define DISASM_BYTES_MAX       4    /* max Z80 instruction length */

typedef enum {
    RTYPE_CODE,
    RTYPE_DATA_BYTE,
    RTYPE_DATA_WORD,
    RTYPE_DATA_STRING,
    RTYPE_GAP,
    RTYPE_ORPHAN    /* byte z80dasm could not decode — invalid or ambiguous opcode */
} RegionType;

typedef struct {
    int           addr;                          /* absolute address (offset + load_addr) */
    int           offset;                        /* byte offset into rom.bin */
    unsigned char bytes[DISASM_BYTES_MAX];       /* raw bytes from rom.bin */
    int           byte_count;                    /* number of valid bytes */
    RegionType    rtype;                         /* CODE, DATA_*, or GAP */
    char          mnemonic[DISASM_MNEMONIC_MAX]; /* from z80dasm or data formatter */
    char          operands[DISASM_OPERANDS_MAX]; /* from z80dasm or data formatter */
    int           operand_addr;                  /* resolved address operand, -1 if none */

    /* populated by annotate_merge() after disasm_range() returns */
    char          label[DISASM_LABEL_MAX];       /* from annotations.ann [label] */
    char          comment[DISASM_COMMENT_MAX];   /* from annotations.ann [comment] */
    char          block[DISASM_COMMENT_MAX];     /* from annotations.ann [block] */
    char          xref[DISASM_XREF_MAX];         /* from annotations.ann [xref] */

    /* populated by symbols_resolve() after disasm_range() returns */
    char          sym_name[DISASM_LABEL_MAX];    /* symbol name if operand_addr matches */
    int           sym_type;                      /* SymbolType enum value, -1 if none */
} DisasmLine;
```

### Field Notes

**addr** — absolute address in the Z80 memory map: `offset + load_addr`.

**offset** — byte index into `rom.bin`. This is the primary key used to look
up annotations and is what `listing.mnm` stores.

**bytes / byte_count** — raw bytes read from `rom.bin` at `[offset,
offset + byte_count)`. Populated by `disasm.c` using `disasm_instr_size()` to
determine `byte_count` for CODE lines, and from the region length for data lines.
z80dasm is not the source of this data.

**mnemonic / operands** — CODE lines start from z80dasm text, then pass through
z80bench normalization (hex literals normalized to `0x` style). Symbol/label
resolution may later rewrite address literals in `operands` to names. For data
lines (`DEFB`, `DEFW`, `DEFM`) text is generated by `disasm.c` directly. For
`RTYPE_ORPHAN` lines the mnemonic is
`DEFB` and operands show the raw byte value; the line is flagged so the UI
can render it distinctly and `xref_build()` can collect it separately.

**operand_addr** — if the operand resolves to a single address (e.g. `JP 0x0050`,
`CALL 0x1000`), this is that integer value. Set to `-1` if the instruction has
no address operand. Used by `symbols_resolve()` for symbol/label resolution and
xref collection.

**label / comment / block / xref** — left as empty strings by `disasm.c`.
Populated by `annotate_merge()` using offset as the lookup key against the
`annotations.ann` data loaded by `annotate.c`.

**sym_name / sym_type** — left as empty string / -1 by `disasm.c`. Populated
by `symbols_resolve()` which checks `operand_addr` against the symbol table from
`symbols.sym`.

## Function Signatures

```c
/*
 * disasm_range()
 *
 * Disassemble [offset_start, offset_end] inclusive, respecting region types.
 * CODE regions are sent to z80dasm; DATA_* and GAP regions are formatted
 * directly. Results are merged in address order.
 *
 * rom           - full ROM blob (rom.bin contents)
 * rom_size      - size in bytes
 * load_addr     - absolute address corresponding to rom[0]
 * offset_start  - first byte offset to process (inclusive)
 * offset_end    - last byte offset to process (inclusive)
 * regions       - region descriptors sorted by offset, from annotations.ann
 * nregions      - count of regions[]
 * labelfile     - path to pre-written z80dasm label file (NULL = omit -l flag)
 * out           - caller-allocated output array
 * out_max       - capacity of out[]
 *
 * Returns number of DisasmLine records written, or -1 on error (errno set).
 *   ENOENT  - z80dasm not found in PATH
 *   EIO     - z80dasm exited non-zero or produced no output for a CODE region
 *   ENOBUFS - out[] too small
 *   EINVAL  - bad range or parameters
 */
int disasm_range(
    const unsigned char *rom,
    int                  rom_size,
    int                  load_addr,
    int                  offset_start,
    int                  offset_end,
    const Region        *regions,
    int                  nregions,
    const char          *labelfile,
    DisasmLine          *out,
    int                  out_max
);

/*
 * disasm_write_mnm()
 *
 * Write a listing.mnm file from a DisasmLine[] array.
 * Called after disasm_range() produces the full ROM listing.
 * The .mnm caches the output so the UI does not re-invoke z80dasm on startup.
 *
 * Returns 0 on success, -1 on error.
 */
int disasm_write_mnm(const DisasmLine *lines, int nlines, const char *path);

/*
 * disasm_read_mnm()
 *
 * Read a listing.mnm file into a DisasmLine[] array.
 * Populates: offset, byte_count, mnemonic, operands, operand_addr.
 * The bytes[] array is filled from the rom blob using offset + byte_count.
 * Annotation fields (label, comment, block, xref) and sym_* are left empty —
 * the caller runs annotate_merge() and symbols_resolve() afterwards.
 *
 * Returns number of lines read, or -1 on error.
 */
int disasm_read_mnm(const char          *path,
                    const unsigned char *rom,
                    int                  load_addr,
                    DisasmLine          *out,
                    int                  out_max);

/*
 * disasm_write_labelfile()
 *
 * Write a z80dasm-compatible label file to `path` combining:
 *   - all [label] entries from ann (offset converted to absolute address)
 *   - all EQU addresses from sym
 * Duplicate addresses are deduplicated; [label] names take precedence.
 *
 * Called by project.c before invoking disasm_range().
 * Returns 0 on success, -1 on error.
 */
int disasm_write_labelfile(
    const AnnData *ann,
    const SymData *sym,
    int            load_addr,
    const char    *path
);
```

## z80dasm Invocation

z80dasm is invoked once per contiguous CODE region in the requested range.

```sh
z80dasm -g <abs_start> -e <abs_end> -l <labelfile> <romfile>
```

- `-g` — absolute start address (`offset_start + load_addr`)
- `-e` — absolute end address inclusive (`offset_end + load_addr`)
- `-l` — label file path; omitted if `labelfile` is NULL
- stdout is captured and parsed line by line
- stderr is discarded
- The ROM is written to a single temp file at the start of `disasm_range()`,
  reused across all region invocations, and deleted before returning

## z80dasm Output Format

z80dasm produces one line per instruction in this format:

```
[label:]\t<mnemonic>[\t<operands>]
```

A label appears on the same line as the instruction when z80dasm generates an
auto-label for a jump target not found in the label file. When a label was
provided via the label file, z80dasm substitutes the name into the operand
field rather than printing the raw address.

**Address tracking** — z80dasm does not print an address column. `disasm.c`
reconstructs addresses by:

1. Starting a running address at the `-g` value
2. Calling `disasm_instr_size(rom, current_offset, rom_size)` to get byte length
3. Advancing `current_offset` by that length after each parsed line
4. `current_offset = current_address - load_addr`

`disasm_instr_size()` is the only Z80-specific knowledge `disasm.c` holds
independently of z80dasm.

## Orphan Byte Detection

When z80dasm cannot decode a byte sequence it emits one of several error
patterns in its output. `disasm.c` must recognise these and emit an
`RTYPE_ORPHAN` line rather than an `RTYPE_CODE` line.

**Known z80dasm error patterns to match (case-insensitive):**

```
defb  NNh          ; invalid opcode
; invalid opcode
; illegal opcode
defb  NNh          ; undefined
```

Any output line from a CODE region that matches one of these patterns, or that
z80dasm emits a bare `defb` for where an instruction was expected, is treated
as an orphan.

**`RTYPE_ORPHAN` line format:**

```
rtype     = RTYPE_ORPHAN
mnemonic  = "DEFB"
operands  = "0xNN"        /* the offending byte value */
byte_count = 1            /* always 1 — only the first byte is tracked */
operand_addr = -1
```

Only the first byte of a run of undecodable bytes is recorded as an orphan —
the address is sufficient for the user to navigate to the problem. Subsequent
garbled bytes in the same run will appear as further orphan lines or gap lines
depending on what z80dasm emits.

Orphan lines are collected by `xref_build()` alongside referenced addresses and
surfaced in the References panel with type `ORPHAN`, allowing the user to sort
and batch-process them.

## Instruction Size Table

`disasm_instr_size()` returns the byte length of the instruction at `rom[offset]`.

```c
int disasm_instr_size(const unsigned char *rom, int offset, int rom_size);
```

Implemented as a two-level prefix lookup:

| First byte | Action |
|------------|--------|
| `0xDD`, `0xFD` | IX/IY prefix: check second byte. If `0xCB`: size = 4. Otherwise use `ix_sizes[second_byte]`. |
| `0xED` | Extended prefix: use `ed_sizes[second_byte]` |
| `0xCB` | Bit prefix: size always = 2 |
| All others | `main_sizes[byte]` — 256-entry table, values 1, 2, or 3 |

`main_sizes[]`, `ed_sizes[]`, and `ix_sizes[]` are static const arrays taken
directly from Z80 documentation. They are small, fixed, and can be verified
against z80dasm's own source.

## Operand Address Extraction

After z80dasm substitutes labels, `operand_addr` is extracted from the operand
string as follows:

1. If the operand string matches a known label or symbol name, look up its address
   in the label/symbol tables — that address is `operand_addr`.
2. If the operand contains a hex literal in z80dasm format (`NNNNh` or `0NNNNh`),
   parse it as the address.
3. Otherwise set `operand_addr = -1`.

`JR` and `DJNZ` relative targets are resolved to absolute addresses by z80dasm
in its output, so the same extraction logic applies.

## Data Region Formatting

For non-CODE regions, `disasm.c` generates `DisasmLine` records directly without
invoking z80dasm. All data region lines have `operand_addr = -1`.

**RTYPE_DATA_BYTE** — one line per up-to-4-byte chunk:
```
mnemonic  = "DEFB"
operands  = "0xAB,0xCD,..."
byte_count = 1..4
```

**RTYPE_DATA_WORD** — one line per two bytes, little-endian:
```
mnemonic  = "DEFW"
operands  = "0x1234"
byte_count = 2
```
Odd trailing byte emitted as a `DEFB` line.

**RTYPE_DATA_STRING** — one line per region:
```
mnemonic  = "DEFM"
operands  = "\"HELLO\""
byte_count = <region length>
```
Non-printable bytes are escaped inside the string representation.

**RTYPE_GAP** — same chunking format as `DATA_BYTE` (`DEFB`, up to 4 bytes/line):
```
mnemonic  = "DEFB"
operands  = "0xAB,0xCD,..."
byte_count = 1..4
rtype     = RTYPE_GAP
```

**RTYPE_ORPHAN** — one line per detected error, always one byte:
```
mnemonic  = "DEFB"
operands  = "0xAB"
byte_count = 1
rtype     = RTYPE_ORPHAN
```
Visually identical to a GAP line in exports, but carries a distinct `rtype` so
the References panel can collect and surface it separately. In the `.lst` export
an orphan line appends `; *** ORPHAN ***` as a comment. In the `.asm` export
it is emitted as a plain `DEFB` with the same comment, which is valid z88dk
syntax.

## Label File Format

z80dasm label files use no `0x` prefix — plain hex addresses:

```
0000 RESET
0050 INIT
1000 CLS
7800 MDATA
```

`disasm_write_labelfile()` converts all `[label]` offsets to absolute addresses
(`offset + load_addr`) and writes them alongside all symbol EQU addresses from
`symbols.sym`. One entry per line, four-digit uppercase hex, space, name.

## Annotation and Symbol Merging

`disasm_range()` returns lines with annotation and symbol fields empty.
The calling code (in `project.c` or `ui_listing.c`) runs two passes:

```c
annotate_merge(lines, nlines, ann);    /* fills label, comment, block, xref */
symbols_resolve(lines, nlines, sym);   /* fills sym_name, sym_type */
```

`annotate_merge()` looks up each line's `offset` in the `AnnData` table.
`symbols_resolve()` looks up each line's `operand_addr` in the `SymData` table.
Both are O(n) passes over the line array with hash or sorted-array lookups.

---

## Error Handling

`disasm_range()` returns -1 and sets `errno`:

| errno | Condition |
|-------|-----------|
| `ENOENT` | z80dasm not found in PATH |
| `EIO` | z80dasm exited non-zero or produced no output for a CODE region |
| `ENOBUFS` | `out[]` capacity exceeded |
| `EINVAL` | `offset_start > offset_end`, or range outside ROM |

The GTK4 UI layer displays an error dialog on non-zero return.

## Testing

`test_disasm.c` — standalone driver:

1. Load a known ROM with a mix of code and data regions
2. Call `disasm_range()` with a hand-crafted label file and region table
3. Call `annotate_merge()` and `symbols_resolve()` with test data
4. Call `disasm_write_mnm()`, then `disasm_read_mnm()`, verify round-trip fidelity
5. Diff code-region mnemonic/operand output against raw z80dasm stdout as a
   correctness check
